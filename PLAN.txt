PROJECT OVERVIEW
This repository is a browser-based C code visualizer built with Vite + React. It provides:
1) A custom C-like tokenizer/parser/interpreter for step-by-step execution (Trace Execution).
2) A memory and state visualizer that renders variables, heap allocations, call stack, and structure-specific views.
3) A local WASM-based C compiler/runtime for Compile & Run using clang/wasm-ld assets shipped in public/wasm.
4) A lightweight analysis tab that detects basic data structures and estimates time/space complexity from the AST.

KEY COMPONENTS
1) UI Shell (React)
   - File: src/App.jsx
   - Editor, tabs (Output, Input, Visualizer, Analysis), and controls.
   - Trace Execution controls with step navigation and play.
   - Input “Scan” button to capture stdin before running.

2) C Tokenizer/Parser/Interpreter
   - File: src/App.jsx
   - Tokenizer: splits C-like code into tokens.
   - Parser: builds an AST for a subset of C.
   - Interpreter: executes AST, records snapshots, handles globals, stack frames, heap, arrays, structs.
   - Step limit to avoid infinite loops and guard failures.

3) Visualization Layer
   - File: src/App.jsx
   - Variable cells, heap cards, call stack cards.
   - Specialized views:
     - Linked list view (circles + arrows)
     - Binary/AVL tree view (tree layout)
     - Stack/Queue view (boxes)

4) Trace Worker
   - File: src/App.jsx (inline worker source)
   - Runs interpreter in a Web Worker to avoid UI blocking.

5) WASM Compiler Runtime
   - File: src/wasmCompiler.js
   - Loads clang/wasm-ld/memfs/sysroot from public/wasm and compiles C to WASM.
   - Executes compiled WASM with WASI stubs and routes stdout to UI.

6) WASM Worker
   - File: src/wasmWorker.js
   - Runs compile/link/execute in a worker and posts stdout back to the UI.

7) Asset Copy Script
   - File: scripts/copy-wasm-assets.js
   - Copies wasm toolchain assets from node_modules into public/wasm.
   - Invoked by postinstall in package.json.

IMPLEMENTATION PLAN
1) Code Input and Editing
   - Use the existing editor and highlight overlay.
   - Keep the current line-number sync and scroll logic.

2) Trace Execution (Interpreter Path)
   - Preprocess code (#define to const int).
   - Tokenize -> Parse -> Interpret.
   - Record step snapshots (globals, stack, heap, stdout).
   - Present steps in the Visualizer tab with step controls.

3) Visualization
   - Render stdout, call stack, globals, heap blocks.
   - Detect pointers and struct fields for linked list/tree drawing.
   - Derive stack/queue from global arrays + indices.

4) Compile & Run (WASM Path)
   - On run, send code + scanned stdin to wasmWorker.
   - wasmWorker compiles, links, executes with wasmCompiler.
   - Capture stdout and display in Output tab.

5) Analysis Tab
   - Parse AST and detect:
     - Linked list, binary tree, AVL tree (field-based).
     - Loop depth for basic time complexity estimate.
     - Array/heap usage for space complexity estimate.
   - Display results in Analysis tab.

6) Input Handling
   - Input tab provides Scan button.
   - Scanned input is locked and used for Trace and Compile.
   - If not scanned, run will auto-capture current input.

FILES AND LOCATIONS
- src/App.jsx: UI, interpreter, visualization, analysis, trace worker.
- src/wasmCompiler.js: WASM toolchain runtime.
- src/wasmWorker.js: Compile/run worker.
- scripts/copy-wasm-assets.js: Asset copy.
- public/wasm: clang, lld, memfs, sysroot.tar.

USAGE FLOW
1) Paste C code in editor.
2) Provide stdin in Input tab and click Scan.
3) Use Trace Execution for step-by-step visualization.
4) Use Compile & Run for real execution output via WASM toolchain.
